策略模式是什么

官方解释

  • 百度百科：策略模式是指有一定行动内容的相对稳定的策略名称。策略模式在古代中又称“计策”，简称“计”，如《汉书·高帝纪上》：“汉王从其计”。这里的“计”指的就是计谋、策略。策略模式具有相对稳定的形式，如“避实就虚”、“出奇制胜”等。一定的策略模式，既可应用于战略决策，也可应用于战术决策；既可实施于大系统的全局性行动，也可实施于大系统的局部性行动。
  • 维基百科中文：策略模式作为一种软件设计模式，指对象有某个行为，但是在不同的场景中，该行为有不同的实现算法。比如每个人都要“交个人所得税”，但是“在美国交个人所得税”和“在中国交个人所得税”就有不同的算税方法。 策略模式： 定义了一族算法（业务规则）； 封装了每个算法； 这族的算法可互换代替（interchangeable）。

个人理解

策略模式，我个人理解的是，为了达到某个目的，在不同的情况下有多种不同的方式，比如说小明写作业，小明很勤快自己写，小明不想思考抄学霸小红的，小明自力更生抄小猿搜题的，小明是懒惰的大富豪花钱雇小刚帮他抄等等，最终都达到了完成作业这个目的。每一种方式都是一种策略。

策略模式的优点

  1. 算法可以自由切换。
  2. 避免了使用多重条件判断。如果不用策略模式，我们可能会使用多重条件语句，不利于代码的维护。
  3. 扩展性良好，增加一个策略只需要实现接口就可以。

策略模式的缺点

  1. 每一个策略都是一个类，每当要增加一个策略就要增加一个类，但是每一个策略的复用的可能性比较小，就会造成代码的臃肿和冗余。
  2. 所有的策略类都需要对外暴露。

策略模式的使用场景

  • 多个类只有算法或行为上稍有不同的场景。
  • 算法需要自由切换的场景。
  • 需要屏蔽算法规则的场景。

策略模式的应用实例

  • 商场促销方式，打折、满减等。
  • Java AWT中的LayoutManager，机布局管理器

策略模式与其他模式的区别

  • 与状态模式的区别

    • 策略模式只是条件选择方法，只执行一次方法，而状态模式是随着状态的改变不停地更改执行方法。比如，我们出去旅游，对于策略模式来讲我们只需要选择一种出行方式就好了，比如自行车、小汽车、火车、高铁、飞机，根据不同情况选择不同方式到达目的地。但状态模式不一样，可能当我们骑自行车到了A地，想坐火车了，就从A地坐火车到了B地，然后又想坐飞机了，就从B地坐飞机到达C地，状态不同选择不同。

  • 与工厂模式的区别

    • 工厂模式是创建型模式，它关注对象创建，提供创建对象的接口，让对象的创建与具体的使用客户无关。策略模式是对象行为型模式，关注行为和算法的封装。比如，还是我们去旅行，对于策略模式我们是根据情况选择一种方式出行，而对于工厂模式，是在我们决定好用哪种方式去旅行之后，由工厂代替我们去构建具体方案，即工厂代替我们去买火车票或者其他什么票。
